# 실전 전환 체크리스트

## 전환 전 확인 (4주 MOCK 테스트 후)

- [ ] Collector 5일 연속 무장애
- [ ] 가상 매수 체결 10건 이상
- [ ] 가상 매도 (손절/트레일링/EOD) 정상 작동
- [ ] 레짐 전환 1회 이상 경험
- [ ] 텔레그램 알림 누락 없음
- [ ] 승률 38%+ (백테스트 대비 -10% 이내)
- [ ] 전문가 회의 (/expert-meeting) PASS

## .env 변경 사항 (MOCK → LIVE)

```
# 모드 전환
OPERATION_MODE=LIVE        # MOCK → LIVE
AUTO_TRADE_ENABLED=true    # 유지

# 시드 30만원 기준
AUTO_TRADE_AMOUNT=100000   # 250000 → 100000 (시드의 33%)
MAX_ORDER_AMOUNT=150000    # 500000 → 150000
MAX_SLOTS=2                # 유지

# 손실 한도 (시드 대비)
MAX_DAILY_LOSS=6000        # 30000 → 6000 (시드의 2%)
MAX_MONTHLY_LOSS=20000     # 100000 → 20000 (시드의 6.7%)
MAX_CONSEC_STOPLOSS=2      # 유지

# 매수 시간
BUY_END_HOUR=15            # 유지
```

## 단계별 증액 계획

| 단계 | 시기 | 시드 | 1회 매매 | 일일 한도 | 조건 |
|------|------|------|---------|----------|------|
| 1단계 | 실전 시작 | 30만원 | 10만원 | 6,000원 | MOCK 4주 통과 |
| 2단계 | +1개월 | 100만원 | 25만원 | 20,000원 | 1단계 수익 양수 |
| 3단계 | +3개월 | 300만원 | 50만원 | 30,000원 | 2단계 승률 40%+ |
| 4단계 | +6개월 | 500~1000만원 | 100~200만원 | 50,000원 | 3단계 연환산 20%+ |

## 주의사항

- 절대 한 번에 큰 금액 투입하지 않기
- 각 단계 최소 1개월 운영 후 증액
- 손실이 월간 한도에 도달하면 해당 월은 중단
- 증액 전 반드시 walk-forward 재최적화 실행 (python -m backtest.walk_forward_auto)
