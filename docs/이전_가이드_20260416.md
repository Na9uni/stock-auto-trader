## 1. Python 경로 문제

### 증상
`run_all.bat` 실행 시 "python.exe를 찾을 수 없습니다" 에러

### 원인
bat 파일에 이전 PC의 Python 경로가 그대로 남아있음

### 확인 방법
```
메모장으로 run_all.bat 열기
→ "C:\Users\이전사용자이름\" 또는 "C:\Python311-32\" 같은 경로가 있으면 문제
```

### 해결
1. 내 PC에 설치된 Python 경로 확인
   - `Win+R` → `cmd` → `where python` 입력
   - 32-bit Python 경로, 64-bit Python 경로 각각 확인

2. 아래 파일들을 메모장으로 열어서 Python 경로 수정
   - `run_all.bat` — 32-bit 경로, 64-bit 경로 둘 다
   - `run_collector.bat` — 32-bit 경로
   - `run_dashboard.bat` — 있으면 확인

## 2. 일봉 데이터 미수집 (시작 후 10분간 전략 안 돌아가는 문제)

### 증상
- 시스템 시작 후 약 10분간 매수 신호가 하나도 안 나옴
- 로그에 "일봉 부족" 경고 (수정 후)
- 수정 전에는 아무 로그도 없이 조용히 무시됨

### 원인
일봉 데이터를 20번째 틱(약 10분)에서야 처음 수집함.
시스템 재시작 시 이전 일봉 데이터가 사라져서 전략 평가 자체가 불가능.

### 해결
`kiwoom/kiwoom_collector.py` 수정 — 첫 틱에서 일봉/계좌/관심종목 즉시 수집

변경 전:
```python
if tick_count % DAILY_EVERY_N_TICKS == 0:
    collector.collect_daily_candles()
```

변경 후:
```python
if tick_count == 1 or tick_count % DAILY_EVERY_N_TICKS == 0:
    collector.collect_daily_candles()
```

Interest, Account 수집도 동일하게 `tick_count == 1 or` 추가.

## 3. 전일 거래량 0 → AI 오판

### 증상
삼성전자처럼 거래량 많은 종목인데도 AI가 "거래하는 사람이 없다"고 판단

### 원인
키움 API의 `opt10001`(주식기본정보)에서 "전일거래량" 필드가 전 종목 0으로 내려옴.
→ 거래량 배수 = 오늘거래량 / 0 = 계산 불가(nan)
→ AI가 "거래 없음"으로 오판

### 해결
`kiwoom/kiwoom_collector.py`의 `_update_stock_data` 함수에 추가:

```python
# prev_volume이 0이면 일봉에서 전일 거래량 보정
if updated["prev_volume"] == 0 and candles_1d and len(candles_1d) >= 2:
    updated["prev_volume"] = candles_1d[1].get("volume", 0)
```

## 4. AI한테 체결강도를 안 넘기는 문제

### 증상
체결강도가 146(사려는 사람 많음)인데도 AI가 모르고 "관망" 판단

### 원인
`signal_runner.py`에서 `ai.quick_signal_alert()` 호출 시 `exec_strength` 파라미터 누락

### 해결
`alerts/signal_runner.py`에서 AI 호출 부분에 추가:

```python
_exec = float(info.get("exec_strength", 0.0))

ai_result = ai.quick_signal_alert(
    ...
    exec_strength=_exec,  # 이 줄 추가
)
```

그리고 `vol_ratio`가 nan일 때 kiwoom_data에서 재계산하는 코드도 추가:

```python
import math
if math.isnan(_sig_vol):
    _cur_vol = info.get("volume", 0)
    _prev_vol = info.get("prev_volume", 0)
    if _prev_vol > 0:
        _sig_vol = _cur_vol / _prev_vol
```

## 5. AI 분석 메시지가 너무 어려운 문제

### 증상
텔레그램 AI 분석이 "체결강도 0.0 및 거래량 배수 nan은 실질적 거래 부재를 의미하며..." 같은 전문 용어

### 해결
`ai/ai_analyzer.py`의 프롬프트를 수정:
- 숫자를 AI한테 그대로 주지 않고, 사람 말로 번역해서 전달
  - 예: `exec_strength=146.78` → "사려는 사람이 더 많음"
  - 예: `vol_ratio=nan` → "거래가 거의 없음"
- AI한테 "초등학생도 알아듣게 쉽게, 전문 용어 쓰지 마" 지시

## 6. MOCK 모드에서 CASH 레짐 차단 문제

### 증상
MOCK(가상매매) 모드인데도 CASH 레짐이면 전략 평가 자체를 안 함
→ 가상 매매 테스트 불가

### 해결
`alerts/signal_runner.py`에서 CASH/DEFENSE 처리 부분 수정:

변경 전:
```python
if regime == RegimeState.CASH:
    _execute_regime_liquidation(data, engine)
    return  # 무조건 중단
```

변경 후:
```python
if regime == RegimeState.CASH:
    _execute_regime_liquidation(data, engine)
    if _OP_MODE != "MOCK":
        return  # LIVE만 중단
    # MOCK: 청산 후 전략 평가 계속
```

## 7. .env 설정 확인

### 필수 확인 항목
```
KIWOOM_MOCK_MODE=True       ← 가상매매 (실전은 False)
OPERATION_MODE=MOCK         ← 가상매매 (실전은 LIVE)
EOD_LIQUIDATION=false       ← 스윙 모드 (데이트레이딩은 true)
```

### 주의
- `.env` 파일은 절대 git에 올리지 않음
- 계좌번호, 텔레그램 토큰 등이 들어있으므로 유출 주의

## 빠른 점검 체크리스트

시스템 시작 후 아래를 순서대로 확인:

| # | 확인 | 방법 |
|---|------|------|
| 1 | bat 파일 Python 경로 | 메모장으로 열어서 내 PC 경로인지 확인 |
| 2 | .env 설정 | MOCK/LIVE, 스윙/데이트레이딩 확인 |
| 3 | 시작 후 첫 로그 | "일봉 수집 OK" 메시지가 나오는지 확인 |
| 4 | 30초 후 텔레그램 | 시스템 시작 메시지 오는지 확인 |
| 5 | 2~3분 후 | 매수 신호 or "거부권 발동" 로그 나오는지 확인 (아무것도 없으면 문제) |
| 6 | AI 분석 메시지 | 전문 용어 없이 쉬운 말로 나오는지 확인 |
